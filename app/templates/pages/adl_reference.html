Manual de Referencia del ADL

Este documento describe las partes que componen el Assistant Description Language (ADL) y cómo utilizarlo para definir un asistente de IA. Incluye:

  • La estructura general del ADL.
  • Las secciones metadata y assistant_instructions.
  • La obligatoriedad y finalidad de cada propiedad.
  • Ejemplos concretos en YAML.

Estructura General del ADL

El fichero ADL consta de dos propiedades principales a nivel raíz:

1 metadata obligatorio
2 assistant_instructions obligatorio
Ambas propiedades son obligatorias para que el ADL sea válido y funcional.

Sección metadata

Tipo: object
Obligatoria: Sí
Finalidad:

Proporciona información descriptiva y de autoría sobre el asistente, incluyendo nombre del autor, organización, licencia, etc.

La sección metadata contiene varias propiedades, de las cuales dos son obligatorias: author y description. Las demás son opcionales pero recomendadas para llevar un historial o definir la visibilidad y los derechos de uso.

metadata.author
  • Tipo: object
  • Obligatoria: Sí
  • Finalidad: Indica el creador del asistente (y, opcionalmente, información de contacto y rol).
  • Propiedades:
    • name (obligatoria, tipo string): nombre del autor.
    • role (opcional, tipo string): rol o puesto que desempeña.
    • contact (opcional, tipo string): información de contacto.
    • organization (opcional, tipo string): institución o empresa asociada.

Ejemplo de uso:

author:
  name: "Jane Doe"
  role: "Desarrolladora IA"
  contact: "jane.doe@example.com"
  organization: "OpenAI Spain"
metadata.description
  • Tipo: object
  • Obligatoria: Sí
  • Finalidad: Proporciona la descripción general del asistente (título, resumen, etc.).
  • Propiedades:
    • title (obligatoria, tipo string): título principal.
    • summary (obligatoria, tipo string): descripción o resumen corto.
    • coverage (opcional, tipo string): finalidad principal del asistente.
    • educational_level (opcional, array de string): niveles educativos sugeridos.
    • use_cases (opcional, array de string): escenarios de uso.
    • keywords (opcional, array de string): palabras clave.

Ejemplo de uso:

description:
  title: "Asistente de Idiomas"
  summary: "Ayuda a practicar conversación y gramática."
  coverage: "English"
  educational_level:
    - "Secundaria"
  use_cases:
    - "Práctica de listening"
    - "Revisión de textos"
  keywords:
    - "idiomas"
    - "educación"
metadata.visibility
  • Tipo: object
  • Obligatoria: No
  • Finalidad: Define si el asistente es público o privado.
  • Propiedad:
    • is_public (tipo boolean)

Ejemplo de uso:

visibility:
  is_public: true
metadata.rights
  • Tipo: string
  • Obligatoria: No
  • Finalidad: Indicar licencias o restricciones de uso.

Ejemplo:

rights: "CC BY-NC 4.0"
metadata.creation_date y metadata.last_update
  • Tipo: string
  • Obligatorias: No
  • Finalidad: Registrar la fecha de creación y la última modificación.
metadata.history
  • Tipo: array de string
  • Obligatoria: No
  • Finalidad: Llevar un historial de cambios o versiones.

Sección assistant_instructions

Tipo: object
Obligatoria: Sí
Finalidad:

Define la lógica de comportamiento, estilo, capacidades y herramientas para el asistente.

Incluye las siguientes propiedades obligatorias:

context style_guidelines final_notes help_text role behavior capabilities tools
assistant_instructions.context
  • Tipo: object
  • Obligatoria: Sí
  • Finalidad: Describir el contexto y manejo de datos del usuario.
  • Propiedades obligatorias:
    • context_definition (array de string)
    • integration_strategy (array de string)
    • user_data_handling (array de string)

Ejemplo:

context:
  context_definition:
    - "El asistente resuelve dudas de programación."
    - "Usuarios de nivel intermedio."
  integration_strategy:
    - "Integración con editor de código online."
    - "API de repositorios de snippets."
  user_data_handling:
    - "No se almacenan datos personales."
    - "Cumple la normativa RGPD."
assistant_instructions.style_guidelines
  • Tipo: object
  • Obligatoria: Sí (aunque sus propiedades internas son opcionales)
  • Finalidad: Fijar el tono y formato de las respuestas.
  • Propiedades opcionales:
    tone level_of_detail formatting_rules

Ejemplo:

style_guidelines:
  tone: "Formal y didáctico"
  level_of_detail: "Detallado con ejemplos"
  formatting_rules:
    - "Usar viñetas para pasos"
    - "Resaltar en negrita palabras clave"
assistant_instructions.final_notes
  • Tipo: array de string
  • Obligatoria: Sí
  • Finalidad: Notas o recordatorios globales.

Ejemplo:

final_notes:
  - "Revisar la ortografía antes de responder."
  - "No ofrecer consejos médicos ni legales."
assistant_instructions.help_text
  • Tipo: string
  • Obligatoria: Sí
  • Finalidad: Mensaje mostrado cuando el usuario solicita ayuda.

Ejemplo:

help_text: "Escribe /help para ver los comandos disponibles."
assistant_instructions.role
  • Tipo: string
  • Obligatoria: Sí
  • Finalidad: Definir la personalidad o rol (por ejemplo, "Asistente Experto").
assistant_instructions.behavior
  • Tipo: object
  • Obligatoria: Sí
  • Finalidad: Describe las reacciones del asistente en distintas situaciones.
  • Propiedades obligatorias:
    • invalid_command_response (string)
    • on_greeting (array de string)
    • on_help_command (array de string)
    • unrelated_topic_response (string)
    • prompt_visibility (string)

Ejemplo:

behavior:
  invalid_command_response: "Lo siento, no reconozco ese comando."
  on_greeting:
    - "¡Hola! ¿En qué puedo ayudarte?"
    - "Encantado de ayudarte hoy."
  on_help_command:
    - "Lista de comandos: /explain, /example, /test."
  unrelated_topic_response: "No tengo información sobre ese tema."
  prompt_visibility: "hidden"
assistant_instructions.capabilities
  • Tipo: array de string
  • Obligatoria: Sí
  • Finalidad: Lista de habilidades principales del asistente.

Ejemplo:

capabilities:
  - "Proporcionar ejemplos de código"
  - "Explicar algoritmos"
assistant_instructions.tools
  • Tipo: object
  • Obligatoria: Sí
  • Finalidad: Define los comandos, opciones y decoradores que el asistente puede usar.
  • Propiedades obligatorias: commands, options
  • Propiedad opcional: decorators
assistant_instructions.tools.commands
  • Tipo: object
  • Obligatoria: Sí
  • Finalidad: Cada comando es un objeto con display_name, description e internal_description.

Ejemplo:

commands:
  explain:
    display_name: "/explain"
    description: "Ofrece una explicación detallada."
    internal_description: "Nivel adaptado al usuario."
  example:
    display_name: "/example"
    description: "Muestra un ejemplo de código."
    internal_description: "Incluye comentarios."
assistant_instructions.tools.options
  • Tipo: object
  • Obligatoria: Sí
  • Finalidad: Parámetros que complementan los comandos.

Ejemplo:

options:
  language:
    display_name: "--language"
    description: "Define el lenguaje de programación (Python, JavaScript, etc.)"
    internal_description: "Por defecto: Python."
  difficulty:
    display_name: "--difficulty"
    description: "Nivel: básico, intermedio o avanzado."
    internal_description: "Por defecto: intermedio."
assistant_instructions.tools.decorators (opcional)
  • Tipo: object
  • Obligatoria: No
  • Finalidad: Añadir "etiquetas" o modificaciones adicionales a la salida.

Ejemplo:

decorators:
  highlight:
    display_name: "@highlight"
    description: "Resalta texto importante."
    internal_description: "Se usa para llamar la atención."
